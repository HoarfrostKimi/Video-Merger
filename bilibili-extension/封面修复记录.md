# 封面竞态修复记录（content.js）

> 目标：彻底解决「封面还没选完/没选上就进下一个视频」。

## 一、根因回顾

`stepSetCover` 全程用「固定 `sleep` + 立即判定/跳过」驱动，B站 慢网、大图、DOM 渲染延迟时，定时器一到就放行，封面其实还没落上。

| 位置 | 原写法 | 风险 |
|------|--------|------|
| 截帧注入后 | `sleep(5000)` 就点「完成」 | 5s 内没传完也放行 |
| `confirmCoverDialog` | `sleep(1500)` 后找「完成」按钮 | 弹窗刚开按钮未渲染 → 找不到 |
| 找上传区 | `sleep(2000)` + 找不到立即跳过 | 弹窗未渲染就误跳 |
| 找封面 input | 找不到立即跳过 | 同上 |

## 二、已落地的改动

### 1. 新增通用 `waitFor(condition, timeout=8000, interval=300)`
轮询等待条件成立，取代零散的固定 `sleep`。超时返回 `null`，调用方再决定降级策略。

### 2. 截帧分支：真状态等待（上一轮已做）
`await waitForCoverUploaded(scope, 30000)` → 预览图加载完成 / 上传中提示消失 才点「完成」。

### 3. `confirmCoverDialog`：等按钮出现
```js
// 旧
await sleep(1500)
const doneBtn = findButtonInScope('完成', scope)
// 新
const doneBtn = await waitFor(() => findButtonInScope('完成', scope), 6000)
```

### 4. `stepSetCover` 截帧分支：等元素挂载
```js
// 旧：固定 sleep(2000) + 找不到即跳过
// 新：等待上传区出现（最多 8s）
const uploadArea = await waitFor(() => Array.from(document.querySelectorAll('*')).find(
  el => el.textContent.includes('上传封面') && el.textContent.includes('拖拽图片') && el.children.length <= 5
), 8000)

// 新：等待封面 file input 出现（最多 5s）
const coverInput = await waitFor(() => {
  const all = document.querySelectorAll('input[type="file"]')
  return Array.from(all).find(inp => (inp.getAttribute('accept') || '').toLowerCase().includes('image'))
}, 5000)
```

### 5. 清理死代码 `skipUpload`（用户确认：所有视频统一截帧封面）
删除 `stepSetCover(file, onLog, skipUpload)` 的参数、注释、`if (skipUpload) 走推荐封面` 分支。

## 三、仍保留的降级分支（刻意不改成死重试）
- 截帧失败（视频解码错误）：硬错误，重试无意义 → 走系统推荐封面 + 提示。
- 找不到「封面设置」按钮：页面结构大变，等待无效 → 提示跳过。

## 四、验证建议
1. 单个视频先跑一遍，在 `?autoFiles=` 触发后观察日志：应出现「封面上传完成，预览图已显示」再点完成。
2. 确认主表单封面缩略图真的换成了截帧图（而非系统默认）。
3. 批量 2~3 个视频，确认每个都等封面落上才进下一个。
4. 弱网卡一次：故意用慢网，验证 30s 兜底不卡死整批。

> 注：`confirmCoverDialog` 目前只校验「弹窗是否关闭」，未校验「封面是否真正应用到主表单」。若线上仍偶发封面没生效，下一步可在此加一道「主表单出现封面缩略图」的回检。
