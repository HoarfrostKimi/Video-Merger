模块严格遵循 Electron 三进程分层：
- `src/main/index.ts` 是主进程唯一入口，负责 `BrowserWindow` 创建、`@electron-toolkit/utils` 初始化 AppUserModelId 与 dev userData、注册全部 `ipcMain.handle` 通道（config/dialog/scan/video/progress/fileServer/browser/control/network/window/app），并内嵌一个本地 HTTP 文件服务器（`http.createServer`）供 Chrome 插件通过 `?fileId=` 拉取已合并的 MP4。
- `src/preload/index.ts` 仅做安全桥接：`contextBridge.exposeInMainWorld('api', ...)` 暴露统一方法集，内部 `invokeApi` 对后端 `{ success, data?, message? }` 响应做自动解包——成功返回 `data`，失败抛错，渲染进程只依赖 `window.api`。
- `src/renderer/src/` 为 React 18 + Vite 渲染层，`main.tsx` 挂载 `<App />`，`App.tsx` 用 Ant Design `ConfigProvider` 管理暗色主题并持久化到本地配置；业务页面集中在 `pages/Home.tsx`。
- 媒体能力封装在 `src/main/ffmpeg.ts`（由主进程导入），通过 `@ffmpeg-installer/ffmpeg` 定位二进制、`child_process.spawn` 启动子进程解析 `-i` stderr 头信息获取时长，并用 concat demuxer + stream copy 拼接多段 FLV 为 MP4；进度通过模块级变量或 `Map<taskId, number>` 暴露给独立的 `progress:get*` 轮询接口。
