- 构建编排：`electron.vite.config.ts` 定义 `main`（Electron 主进程）、`preload`（IPC 桥接）与 `renderer`（React + Ant Design）三个入口，共享同一份 Node 依赖并通过 `externalizeDepsPlugin` 在打包时外置；`tsconfig.json` 通过 Project References 引用 `tsconfig.node.json`（主/预加载侧）和 `tsconfig.web.json`（渲染侧），实现 Node 与浏览器两套类型环境的物理隔离。
- 子模块分工：`app_source` 提供 Electron 主进程与 React 渲染端源码；`tests` 基于 Vitest 直接断言这些纯函数（配置合并、文件名解析、文件分组、FFmpeg 输出解析、IPC 结果解包）的输入输出，绕过 UI 层；`bilibili_extension` 作为独立的 Chrome Manifest V3 扩展，不参与 Electron 构建链路但复用根目录的 npm 脚本约定；`deliverables` 仅产出 PRD/架构评审等文档，不参与任何构建或测试命令。
- 运行时边界由 Electron IPC 契约维系：主进程经 preload 暴露给 renderer，tests 子模块则直接调用同构纯函数进行行为验证，形成「UI 层 + 无头测试」双轨验证。
- 分发流程：`package.json` 的 `scripts` 是跨子模块的统一入口——`dev/build/preview` 走 electron-vite，`dist/pack` 串联 build 与 electron-builder，`test` 直接驱动 tests 子模块用例，`postinstall` 自动安装 ffmpeg 平台依赖。
