本模块为纯文档型交付目录，按软件开发生命阶段组织：
- `架构评审`：对 Electron 33 + React 18 + fluent-ffmpeg 的三段式进程模型进行逐行审计，输出 Strengths/Weaknesses/Improvements 清单（W2 同步扫描阻塞、W3 sandbox:false + userData 误置安装目录、W5 测试镜像副本零覆盖等），并给出 I1–I16 改进建议与优先级。
- `增量 PRD`：以评审结果为依据，将改进项收敛为 A/B/C 三类需求池（安全/性能/代码质量），按 P0/P1/P2 分级，明确 Open Questions（sandbox 可行性、userData 迁移、strict 范围、electronAPI 暴露收敛）与 Out of Scope 边界。
- `增量设计`：配套 Mermaid 类图与序列图，描述关键路径（Home.tsx → preload api → main IPC → ffmpeg.ts → FFmpeg 子进程）及改动点映射。
- `QA 验证`：用于逐项验收上述改进项的回归检查清单。
依赖方向单向：评审 → PRD → 设计 → QA，不引入任何可执行代码或构建产物。
